Information for me : 
Backend - 
id is the primary key of the current table.

For example, in your ShareLink model:

model ShareLink {
  id        String @id @default(cuid())
  pageId    String
  token     String @unique
  access    AccessLevel

  page Page @relation(fields: [pageId], references: [id])
}

Here:

id → identifies a ShareLink
Every share link has its own unique id

Example:

ShareLink
---------
id      = sl_123
pageId  = page_456
token   = abcxyz
pageId

pageId is a foreign key.

It stores the ID of the page that this share link belongs to.

Page
----
id = page_456
title = "Meeting Notes"
ShareLink
---------
id      = sl_123
pageId  = page_456
token   = abcxyz

Notice:

sl_123 identifies the share link
page_456 identifies the page
